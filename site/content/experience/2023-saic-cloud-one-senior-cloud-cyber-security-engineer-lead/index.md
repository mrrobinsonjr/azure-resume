Mission / Scope

Serve as the senior cybersecurity engineer for the Cloud One Common Computing Environment—a globally distributed hybrid DoD cloud hosting mission applications for USAF and Army customers. Responsibilities include leading vulnerability management, continuous monitoring, RMF implementation, and security integration across multiple cloud impact levels.

What I Owned

- Vulnerability identification, triage, and remediation planning across VM, container, and cloud-native platforms.
- Development and maintenance of A&A artifacts: Security Assessment Plans (SAP), Security Assessment Reports (SAR), and POA&Ms supporting RMF steps and ATO sustainment.
- Definition and documentation of inheritable cloud and platform controls aligned to NIST 800-53, DoD SRGs/STIGs, and organizational baselines.
- Integration of security gates and automation into CI/CD pipelines to catch misconfigurations earlier.

Impact Highlights

- Strengthened overall cybersecurity posture across hybrid environments by improving inventory, controls inheritance, and automated checks.
- Reduced ATO friction and audit time by delivering high-quality authorization artifacts and reusable control documentation.
- Improved vulnerability response cadence via automation and expert remediation guidance.

Tooling / Platforms

- ACAS, Nessus, HBSS
- eMASS for A&A artifact management
- Azure, AWS, OCI
- Terraform, Jenkins, GitHub Actions, Azure DevOps
- Python, Bash, PowerShell

Selected Deliverables

- SAP, SAR, and POA&M documentation tailored to Cloud One platforms.
- Control inheritance matrices and reusable control packages for cloud families.
- CI/CD security automation scripts and remediation playbooks.
